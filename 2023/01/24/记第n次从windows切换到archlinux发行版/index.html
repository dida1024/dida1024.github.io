记第n次从windows切换到archlinux发行版

dida
2023-01-24
linux › windows
arch, linux

序言

不知不觉使用了一段时间的windows又开始卡顿，当即决定回到linux的怀抱
在这台电脑已经体验了ubuntu、manjaro并深入使用了一段时间，最终也是因为各种各样的原因又回到了windows

在这次的使用，c盘也逐渐满了，系统也有肉眼可见的卡顿，正好近期使用到windows中的wsl2和docker，其中docker的启动速度让我深受其扰，于是便打算一不做二不休干脆装linux算了

为什么是arch

首先也是因为有一定经验，在台式电脑上使用了一个月左右的archlinux，先是在笔记本上使用了manjaro发行版，感觉就是香但是差点味道，可能是心理作用

使用了arch之后，首先是arch Wiki，牛，就很牛

至于为什么不用Ubuntu，emmmm,首先是莫名的崩溃，虽然不碍事但是不爽，还有安装软件需要导入私钥？？？

然后就是pacman和aur，里面有足够日常使用的软件

我的windows下有什么

如果要迁移系统首先要弄清楚在当前系统下操作的工具和在另一个系统上的代替工具

  • 办公
    • wps(处理文档类型的文件或者查看pdf)
    • 编程
      • idea(java项目)
      • Dbeaver(数据库可视化工具)
      • Another Redis Desktop(redis可视化工具)
      • PDManer(创建数据库工具)
      • WebStorm(前端项目)
      • vsCode(通常编辑器使用)
      • ApiFox(api管理软件)
    • chrome(常用浏览器)
    • 微信(需求性不高)
    • Utools(常用工具，用于打开软件及翻译截图)
    • Clash(看看世界)
    • Telegram(摸鱼的时候看)
    • 网易云音乐(常用音乐播放器)
    • FDM(常用下载器)
    • potplayer(视频播放器)
  • 游戏
    • MC(唯一的经常可能会打开的游戏)
  • 服务
    • mysql
    • docker
      • redis
      • rabbitmq
    • java

我个人的打开电脑的大部分时间花费在使用浏览器和编程上，所以对于平台反而没有强依赖性

服务在linux上只会运行的更顺畅

在arch上常用软件的替代

  • 办公
    • 视频播放器
      • vlc(虽然它很好用但是我经常感觉用的没有potplayer舒服)
    • 网易云音乐
      • 选择第三方的播放器(官方的虽然bug不多也好解决，但是更新慢)
    • utools(暂时没找到代替的，官方的utools在linux下呼唤出来总是很别扭)
    • 微信(有解决方法但是有bug，暂时不考虑在笔记本上登陆微信，可以考虑qq，刚出的qq官方版本)
  • 游戏
    • 有mc，但是没有windows稳定

arch桌面的选择

linux系统的桌面可以自己选择喜欢的

  • kde
  • gnome
  • xface
  • i3
  • …….

之前都是用的gnome，这次使用kde

备份windows上的必要数据

  • 代码
    • 直接将数据盘压缩放进移动硬盘备份所有数据
  • 数据库
    • 导出所有数据库包括数据
  • idea插件
    • 备份所有插件名

  • google插件
    • 通过google云备份
